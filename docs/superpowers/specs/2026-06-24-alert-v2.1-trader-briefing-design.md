# Alert v2.1 — Trader's Briefing redesign (+ dynamic trend-line bug fix)

**Date:** 2026-06-24
**Status:** Approved for implementation
**Scope:** `lambdas/shared/narration.mjs` (`composeRichMessage` + new pure helpers)
+ tests. No scanner change (Thesis + Size both kept). No scoring/gate/threshold/
weight/`STRATEGY_VERSION` change. Presentation only — except it also FIXES a real
correctness bug (the hardcoded trend line).

## Two things at once

1. **Redesign** the single-signal alert into a professional trader's briefing:
   quality tier, Gerchik Level front-and-center (support touches + target type),
   1-decimal factor scores, factor-derived "Why It Scored High", tighter sections.
2. **Fix the trend-line bug** (folded in because the redesign rewrites that section):
   `narration.mjs:268` hardcodes `"Trend: above 50MA & 200MA ✓"` unconditionally,
   but the gate only guarantees `close > ma200`. 6/29 of the 2026-06-23 candidates
   (AMT, AMZN, CTVA, NEE, OXY, WBD) are below their 50MA — the line lies for them.
   The redesigned line is DERIVED from `md.close/ma50/ma200`.

## New pure helpers

### `qualityTier(score)` → `{ letter, emoji }`
- `>=80` → `A+` 🅰️ · `70–79` → `A` 🅰️ · `60–69` → `B` 🅱️ · `<60` → `C` 🅲
- Header renders `${emoji} Tier ${letter}`. Score-derived (NOT the watchlist `qualityTier`).

### `trendText(md)` → string (the bug fix)
- `close>ma50 && close>ma200` → `Above 50MA & 200MA ✓`
- `close>ma200` (below 50MA) → `Above 200MA ✓ · below 50MA ⚠️`
- `close>ma50` (below 200MA) → `Above 50MA ✓ · below 200MA ⚠️` (defensive; gate prevents)
- else → `Below 50MA & 200MA ⚠️`
- non-finite inputs → `data unavailable`

### `targetTypeBadge(targetType)` → string
- `RESISTANCE` → `🟢 Resistance` · `PROJECTED_ATR` → `🟡 Projected ATR`
- `RESISTANCE_FLOORED_BY_PROJECTED_ATR` → `🟠 ATR Floor` · else → `—`

### `buildWhyScoredHigh(result, md, extras)` → string[] (only firing bullets)
- RS Rank: `>=90` "Top-decile RS Rank (X)" · `>=80` "Top-quintile RS Rank (X)" · `>=70` "Strong RS Rank (X)"
- EPS YoY: `>=50` "Exceptional EPS growth (+X%)" · `>=20` "Strong EPS growth (+X%)"
- Revenue YoY: `>=20` "Strong revenue growth (+X%)"
- Sector %: `>=70` "Strong sector participation" · `>=50` "Constructive sector strength"
- Trend: `close>ma50 && close>ma200` "Price above 50MA & 200MA" · elif `close>ma200` "Price above 200MA"
- Volume: `vol/avg >= 1.2` "Elevated volume confirms interest"
- Fallback when none fire: "Passed all entry gates"

### `buildRiskBullets(result, md, config)` → string[]
- RSI: `>70` "Overbought (RSI X)" · elif `<55` "RSI momentum is only moderate"
- Target: `PROJECTED_ATR` "No nearby chart resistance available" ·
  `RESISTANCE_FLOORED_BY_PROJECTED_ATR` "Nearest resistance very close (target uses ATR floor)"
- Earnings `<=15d` → "Earnings in X days" · no support → "No nearby support below price"
- Always → "ATR-based stop may gap through overnight"
