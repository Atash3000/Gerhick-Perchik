# gp-2.0.0 — Tier 2 factors as gradient score components

**Date:** 2026-06-22
**Status:** Approved and implemented on branch `gp-2.0.0-tier2-gradient-factors`
(PR open, awaiting human merge). Live `gp-config` row update sequenced post-deploy.
**Strategy version:** `gp-1.0.0` → `gp-2.0.0` (single bump, human-approved, bumped only when the diff is approved)

---

## 1. What this is

Activate four factors the pipeline already captures but ignores in scoring — RS
rank, EPS growth, revenue growth, sector strength — as **neutral gradient
contributions** to the 0–100 score. This is the "stop ignoring good data" move.

It is **not tuning.** Weights are provisional and neutral; they will be *earned*
from accumulated `gp-outcomes` later (Phase B). Per CLAUDE.md this requires a
`STRATEGY_VERSION` bump. **Do not POOL `gp-1.0.0` and `gp-2.0.0` outcomes into a
single performance statistic** (a pooled win-rate across versions is meaningless);
`/analyze` should, however, support comparing them **side-by-side with clear
version labels**.

## 2. Hard rules (non-negotiable, from the approved brief)

- **No new gates.** RS, fundamentals, and sector are *components*, never gates.
- **Missing data is always neutral (0 points), never a rejection or `NO_DATA`.**
  A name with no fundamentals/RS/sector must still be fully scorable on its
  price-action factors. **0 points, NOT a midpoint fallback** (a midpoint
  suggestion from external review was rejected): the score is evidence-weighted,
  so *absence of evidence must contribute zero*, not half-points. A midpoint would
  (a) award unearned points for missing data, (b) rank no-data names above
  genuinely-weak-but-measured names, (c) let names clear `buyScoreThreshold` partly
  on fictional points, and (d) destroy `/analyze`'s `componentEdge` measurability
  by making no-data names indistinguishable from real-midpoint names. "Not
  rejected" means the name stays scorable on price action — it does **not** mean it
  is handed free factor points. *(This is distinct from the `empiricalEdge`
  placeholder at §3, which sits at its midpoint because it is a deliberate,
  every-name Phase-8 stand-in for a known-future factor — not an absent measurement
  for an individual name.)*
- **All existing gates unchanged:** price > 200MA, R:R ≥ min, target > price, no
  HIGH news, earnings not within 3 days, SPY not below 200MA, correlation cap.
- **Observe mode unchanged. `alertMode` untouched. Not going live.**
- **One `gp-2.0.0` bump for all four factors** — not four bumps — so the
  comparable outcome pool isn't fragmented. `gp-1.0.0` history is preserved; its
  outcomes are never pooled into a combined statistic with `gp-2.0.0`, though
  `/analyze` may show the two versions side-by-side under explicit labels.
- **`breakdown` must carry the three new components** (`rsRank`, `growthQuality`,
  `sectorStrength`) so `/stats` and `/analyze` can later measure each one's
  predictive edge. This is what makes Phase B possible — required, not optional.

## 3. Weight re-budget

| Component        | gp-1.0.0 | gp-2.0.0 | Change |
|------------------|---------:|---------:|-------:|
| empiricalEdge*   |       30 |   **15** |   −15  |
| setup            |       20 |       20 |    —   |
| trend            |       20 |   **15** |    −5  |
| momentum         |       15 |   **10** |    −5  |
| volume           |       10 |    **8** |    −2  |
| news             |        5 |    **2** |    −3  |
| **rsRank**       |        — |   **12** |   new  |
| **growthQuality**|        — |   **13** |   new  |
| **sectorStrength**|       — |    **5** |   new  |
| **Total (max)**  |      100 |  **100** |    ✓   |

The four downward-rebudgeted existing components (trend, momentum, volume, news)
free 15 points; empiricalEdge's max drops 15 more; the 30 freed points fund
rsRank 12 + growthQuality 13 + sectorStrength 5 = 30. **New max total = 100.**

\* **empiricalEdge is a fixed-neutral placeholder until Phase 8.** It sits at the
**midpoint of its weight: 7.5 of 15**, so when real outcome data fills it later it
can move a name *up toward 15* or *down toward 0* — symmetric headroom, which is
what "neutral until we know" means. Pinning it at 15 would make it an unearned
flat +15 baseline with only downside; that is rejected. Because it never reaches
its 15 max today, the **achievable ceiling is 92.5, not 100.** (This matches the
prior convention — gp-1.0.0's 30→15 was also exactly half.)

### Threshold re-center: `buyScoreThreshold` 60 → 53 (provisional, mechanical)

**Decision (human-approved): change `buyScoreThreshold` from 60 to 53 in this same
bump.** This is a **structural re-center, NOT outcome tuning** and **NOT a claim
that 53 is validated** — both 60 and 53 are provisional.

**Why 53:** the dominant score shift from gp-1.0.0 to gp-2.0.0 is `empiricalEdge`
moving from a fixed 15 to its honest midpoint 7.5 — a flat **−7.5 for every name**
— only partly offset by the modest new-factor points. The threshold drops ~7 to
preserve gp-1.0.0's selectivity given the budget moved; it does not reflect any
observed win-rate.

**Evidence (offline analysis of the accumulated `gp-snapshots`, 2026-06-22):**
- Universe = 43 names, **one scan day on record**, on which **all 43 were
  gate-rejected** (16× R:R<min, 15× price≤200MA, 12× no target). **Zero candidates
  have ever existed**, so 53 is calibrated against **ZERO actual candidate data**.
- On the 16 names that passed every gate except R:R (the closest-to-signal
  population), the exact v1→v2 score delta had **median −7.3, mean −7.6** (setup
  cancels, so this is exact), 14 of 16 deflating.
- `growthQuality` is null for **30% (13/43)** of the universe (Finnhub lacks both
  EPS and revenue quarterly growth on the same 13 names), but missing fundamentals
  contributes only ~1 extra point of drop — the −7.5 `empiricalEdge` shift
  dominates, not the missing data.

**This is a structural re-center, not an observed one. Phase B MUST revisit the
threshold against real gp-2.0.0 candidates across multiple, varied market days
before it can be considered calibrated.** A price-action-perfect name with no
fundamentals/RS still forfeits up to 30 points by design (missing = neutral-0); 53
does not paper over that — it only re-centers for the deterministic `empiricalEdge`
shift.

## 4. Factor activation rules

All three new scorers live in `scoring.mjs`, are pure, and return **0 on any
missing/non-finite input** (never reject, never `NO_DATA`).

### rsRank (0–12)
- Source: `marketData.rsRank` (already computed cross-sectionally by the scanner's
  `rankPercentiles`, 1–99).
- `points = round(12 · clamp(rsRank, 1, 99) / 99, 2)`. Smooth, no cutoff.
- null/non-numeric → 0.
- Note: cross-sectional across only ~43 names → coarse/noisy by design; the modest
  weight reflects that.

### growthQuality (0–13, combined)
- Source: `marketContext.fundamentals.epsGrowthQtr` and `.salesGrowthQtr`
  (quarterly YoY). EPS and revenue growth are correlated, so they are **combined
  into one component to avoid double-counting.**
- **Clamp** each present metric to **[−100%, +100%]** (`GROWTH_CLAMP_LO/HI`) so an
  extreme Finnhub reading (loss-to-profit ~9000%, or a `$0.01→$0.02` 100% noise
  reading) can't dominate.
- Average whichever metrics are present (one missing → use the other).
- **Smooth ramp**, full at `GROWTH_SAT = +50%` YoY:
  `points = round(13 · clamp(avg, 0, 50) / 50, 2)`. No 25% cliff — pure gradient.
- Both missing → 0.
- Bound check: a 9000%/9000% reading clamps to 100/100 → avg 100 → saturates at
  **13** (the bucket max, modest), so garbage tops out instead of dominating.

### sectorStrength (0–5)
- Cross-sectional, computed in the scanner via a new `sectorStrengthPercentiles`
  in `rs.mjs`: mean `rsRaw` per sector, sectors ranked into 1–99.
- **Sectors with fewer than 3 names (`minNames = 3`) are excluded** — too few to
  be a real signal — and absent from the map; the scanner passes `null` for those
  names → neutral 0. (43 names is noisy → low weight by design.)
- Scoring: `points = round(5 · clamp(pct, 1, 99) / 99, 2)`. Missing/undersized → 0.

## 5. Wiring (data flow)

- `rsRank`: already attached to `marketData` by the scanner (pass 1). Scoring
  reads `marketData.rsRank`. No new plumbing.
- `growthQuality`: scanner already fetches `fundamentals` per name but only stores
  it. **New:** pass `fundamentals` into `score()` via `marketContext.fundamentals`.
- `sectorStrength`: **New:** scanner computes `sectorStrengthPercentiles(...)`
  across the gathered universe (analogous to the existing `rsRank` step) and
  passes the per-name `sectorStrengthPct` via `marketContext`.

`score()` stays pure and backward-safe: all three inputs default to neutral 0 when
absent. The new components are computed **only after every gate passes**, so they
cannot influence gating.

## 6. Components/consumers already generic (verified against main)

## 8. Tests (required)

- **Worked example (fully loaded):** `rsRank 88`→11, fundamentals
  `eps 40 / sales 30` (avg 35)→9, `sectorStrengthPct 75`→4, plus empiricalEdge 7.5,
  setup 16, trend 15, momentum 10, volume 8, news 2 → **score 82.5**. Assert the
  breakdown values **sum to `r.score`** (dynamic sum, not a hardcoded constant
  prone to silent drift if a weight changes).

  (sales present, uses sales); null revenue (eps present, uses eps); both null
  (`growthQuality === 0`); null `sectorStrengthPct` (`sectorStrength === 0`). Each
  still returns a decision — never `NO_DATA`, never a rejection.
- **Extreme-fundamental clamp:** `eps 9000 / sales 9000` → `growthQuality === 13`
  (capped, doesn't dominate).
- **Gate regression:** existing gate tests kept verbatim — gates behave
  identically (they assert gate booleans + NO_SIGNAL/NO_DATA, independent of the
  new components).
- **`rs.mjs`:** sectors ranked by mean `rsRaw`; an undersized sector (<3 names)
  absent from the map (→ neutral); a null-sector item ignored.
